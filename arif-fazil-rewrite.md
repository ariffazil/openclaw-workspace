## G. DESIGN DIRECTION

### Font Pairing Recommendation
- **Display/Headings:** `"Space Grotesk"` — geometric, technical, not generic (already in use, keep it)
- **Body:** `"Inter"` — neutral and readable, acceptable at small sizes (already in use, keep it)
- **Mono/Code:** `"JetBrains Mono"` — appropriate for arifOS tool references (already in use, keep it)
- **Consider adding:** `"Crimson Pro"` or `"Lora"` for theory/writings section — editorial serif for long-form reading contrast

### Palette Recommendation
**Keep the current earth/gold direction. Simplify the naming.**

```
--bg:           #0C0E12   /* deep charcoal — primary surface */
--bg-secondary: #141620   /* elevated surface — cards, panels */
--bg-tertiary:  #1A1D26   /* subtle depth layer */
--border:       #2A2E3A   /* structural lines */
--text:         #E8E4DF   /* warm white — primary text */
--text-muted:   #8B8880   /* warm gray — secondary text */
--gold:         #C9A84C   /* controlled gold — one accent only */
--gold-dim:     #8B7440   /* subdued gold — borders, hover states */
--red:          #8B3A3A   /* muted — used only where structurally necessary */
```

**What to eliminate:**
- Remove `blood: #6B2020` as a primary — it's too heavy for a credibility-first page
- Keep red as an accent ONLY for the most critical alert states
- The cyan (`#00D4AA`) should be retired — it was a design-system borrowed aesthetic

**Single accent rule:** Gold (`--gold`) is the only color that should carry visual emphasis. Everything else is structural.

### Spacing & Mood Rationale
- **Generous vertical rhythm** — sections breathe, not compressed
- **Left-align body text** — center only for the name in the hero
- **Max content width: 720px** — editorial column, not marketing wall
- **Section spacing: 96px desktop / 64px mobile**
- **Card padding: 32px** — comfortable reading, not button-padding density

### What to Avoid Visually
- ❌ Purple/blue AI gradient (obvious slop)
- ❌ Three-column feature grid (startup template aesthetic)
- ❌ Icon-in-colored-circle cards (enterprise SaaS pattern)
- ❌ Centered hero with overlay text on abstract imagery
- ❌ Stat counters with large numbers and no context ("13 FLOORS", "17 TOOLS" as hero elements)
- ❌ Full-width colored sections as visual breaks (use whitespace, not color)
- ❌ Animated background elements (distracting, reduces credibility signal)

### Structural Principles
- **One column of content, max 720px** — reads like editorial, not a dashboard
- **Cards for discoveries** — each discovery is a card, not a bullet list
- **Horizontal rule between major sections** — subtle, structural
- **No sticky nav** — the page should be read, not navigated around
- **Footer is simple** — disclaimer, links, nothing clever
