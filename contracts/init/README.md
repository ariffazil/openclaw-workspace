# AAA 000 INIT Contract

This directory defines the canonical ignition contract for AAA.

## Rule

- `arifos.init` remains the doctrinal boot law.
- `contracts/init/000-init.yaml` is the machine-readable AAA init contract.
- `000_INIT.md` is explanatory support material, not the canonical source of runtime truth.

## Invariant

No AAA runtime may start meaningful work until the 000 contract has restored:

1. identity
2. user anchor
3. temporal anchor
4. spatial anchor
5. continuity state
6. constitutional authority state
7. hold conditions for unsafe ignition
