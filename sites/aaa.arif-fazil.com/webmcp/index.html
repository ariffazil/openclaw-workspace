Browser Transport

WebMCP

Run MCP servers directly in the browser. WebSocket-based transport for web-native AI tool integration.

Why WebMCP?

Traditional MCP uses stdio or SSE transports, requiring a server process. WebMCP brings MCP to the browser, enabling:

  • → Client-side MCP without server infrastructure
  • → Web-based AI assistants with tool access
  • → Reduced latency for browser-based applications
  • → Simplified deployment for web apps

WebSocket Protocol

Connection

WebMCP uses WebSocket as the primary transport. Fallback to HTTP long-polling is supported. 

const ws = new WebSocket('wss://api.arifos.arif-fazil.com/webmcp');

ws.onopen = () => {
  ws.send(JSON.stringify({
    type: 'initialize',
    protocolVersion: '2025-03-26',
    capabilities: { tools: {}, resources: {} }
  }));
};

Message Format

All messages are JSON-encoded with a standard envelope. 

// Request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "verify_constraint",
    "arguments": {
      "content": "Text to validate",
      "floor_id": "F1"
    }
  }
}

// Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [{
      "type": "text",
      "text": "{\"compliant\": true, \"score\": 0.98}"
    }]
  }
}

JavaScript SDK

npm install @arifos/webmcp 
import { WebMCPClient } from '@arifos/webmcp';

const client = new WebMCPClient({
  endpoint: 'wss://api.arifos.arif-fazil.com/webmcp',
  apiKey: 'your-api-key'
});

await client.connect();

// Call a tool
const result = await client.callTool('verify_constraint', {
  content: 'Text to validate',
  floor_id: 'F3'
});

console.log(result.compliant); // true
console.log(result.score); // 0.98

Security Considerations

API Key Authentication

All WebMCP connections require valid API keys. Keys are passed in the connection handshake and validated before any tool execution.

9 Floors Enforcement

Every tool call is validated against the 9 Constitutional Floors. Violations are blocked and returned with detailed explanations.
