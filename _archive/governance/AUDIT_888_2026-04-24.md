## 3. Critical Finding: arifosmcp /health Broken

**Root Cause:** FastMCP's `StarletteWithLifespan` wrapper does not expose `list_tools()`. The `/health` endpoint in `rest_routes.py:1735` calls `await mcp.list_tools()`, which throws:

```
AttributeError: 'StarletteWithLifespan' object has no attribute 'list_tools'
```

The constitutional kernel's panic handler catches this and returns:

```json
{"status":"void","tool":"kernel_panic_handler","error_message":"F13: System halt due to unhandled kernel exception.","action":"HALT"}
```

**Why Docker Shows Healthy:** The Docker healthcheck hits `/sse`, not `/health`. `/sse` works correctly.

**Fix Status:** Commit `d507eadc1` in arifOS repo fixes this. It is **unpushed**.

**Impact:** Any external monitor or Forge session checking `/health` will think the server is down, even though SSE and tool routes may still function.
