**Core thesis acknowledged:**
- MAP = spatial operating system for GEOX, not a map viewer
- Without MAP, WELL/PROSPECT/EARTH3D/TIME4D/CROSS stay fragmented by geometry and interface
- Build to create one truth-bearing spatial substrate; stop duplicating mapping logic; turn tools into workflows; make provenance visible

**Tiered ingestion policy adopted:**
- Tier 1 (always live): USGS earthquake GeoJSON, Natural Earth boundaries/coastlines — already working
- Tier 2 (operational geoscience): MacroSTRAT geologic units, GEBCO bathymetry, USGS active faults, SRTM hillshade, Natural Earth phys
- Tier 3 (strategic/heavy): IRIS seismic metadata, high-res bathymetry, mining overlays, population grids
- Rule: federate don't mirror; prefer BBOX queries, tiles, WMS, STAC/COG patterns

**MAP/EARTH3D separation enforced:**
- MAP = 2D AOI selection, overlays, evidence, fast querying
- EARTH3D = 3D volume, subsurface interpretation

**GEOX dimension tags to use:** map, well, prospect, section, earth3d, time4d, physics, cross

**Implementation objectives (A-E) from co-architect:**
- A: Fix MacroSTRAT geologic units (point-based lat/lng query, not bbox polygon fill)
- B: Implement layer contracts (JSON registry schema, layer panel reads from registry)
- C: Add missing Tier 2 layers (GEBCO, faults, SRTM, Natural Earth)
- D: Wire click→context MCP chain, polygon→AOI MCP chain
- E: Real coordinate transformation with explicit EPSG surfaces

**Build sequence:** layer contracts → MacroSTRAT fix → GEBCO → faults → click/probe chain → polygon/AOI → provenance panel

**Layer contract schema fields:** id, name, tier, dimensions[], source, source_url, license, status (live/partial/blocked), retrieval_method, last_fetch, crs, style_conf, provenance, mcp_tools[], fail_state

### arifOS MCP Server Fix — PR #309

**Fix commits (merged in PR #309):**
- `916aac7` — fix/verdict_wrapper: add missing ExecutionStatus/GovernanceStatus/ArtifactStatus imports
- `0d4de60` — fix/naming: arifOS_kernel → arifos_kernel

**Merge commit:** `59b6f52`

**Server was still running old SHA 909c4ca** after merge — needed pull + rebuild.

**New root cause found:** `arifosmcp/stdio_server.py` in source was a **0-byte empty directory** (created Apr 10). Docker's `COPY . .` captured it, `pip install` tried to copy it as a file and failed. Fixed by:
- Removing `/root/arifOS/arifosmcp/stdio_server.py` (empty directory)
- Added `.dockerignore` with `stdio_server.py`, `__pycache__`, `*.pyc`, etc.

**VAULT999 symlink issue:** `arifosmcp/VAULT999` is a symlink to `/root/VAULT999`. Docker COPY follows symlinks, then `RUN mkdir -p VAULT999` fails because the path already exists as a symlink (not a directory). Fix: `RUN rm -rf VAULT999` before `mkdir -p VAULT999`.

**arifosmcp image vs mounted source conflict:**
- Image has `arifosmcp` pip-installed into site-packages with old buggy `server.py`
- Container bind-mounts `/root/arifOS → /usr/src/project`
- Python loads from site-packages first (path ordering), ignoring mounted source
- **Result:** even with fix committed to source, container still loads old `server.py` from image
- **Confirmed by:** `docker exec arifosmcp python3 -c "from arifosmcp.runtime.server import app; print(app)"` → printed `None`
- **Confirmed fixed after image rebuild:** prints `<FastAPI...>` object

**Dockerfile fix applied:**
```dockerfile
RUN mkdir -p telemetry data memory static/dashboard && rm -rf VAULT999
RUN mkdir -p VAULT999 memory static/dashboard && \
    mkdir -p /ms-playwright && \
    chown -R arifos:arifos /usr/src/app /ms-playwright
```
