## Missing REST Routes тАФ Identified, Not Yet Fixed

**Problem**: Landing page links 10+ routes that return 404 from uvicorn:
- `/.well-known/agent.json` тЖТ 404
- `/.well-known/mcp/server.json` тЖТ 404
- `/.well-known/oauth-authorization-server` тЖТ 404
- `/.well-known/jwks.json` тЖТ 404
- `/api/auth/authorize` тЖТ 404
- `/api/auth/token` тЖТ 404
- `/discovery` тЖТ 404
- `/ready` тЖТ 404
- `/api/governance-status` тЖТ 404
- `/status` тЖТ 404

**Root cause**: `rest_routes.py` does NOT have `@route()` decorators тАФ it only has `mcp.custom_route()`. FastMCP 3.x `custom_route()` requires `methods=["GET"]` kwarg. The route registration was failing silently.

**Fix needed**: Replace `mcp.custom_route(f"/path")` with `mcp.custom_route(f"/path", methods=["GET", "POST"])` or similar in `rest_routes.py`. Additionally:
- FastMCP 3.x may not expose `mcp._app` тАФ need to verify by checking `hasattr(mcp, "_app")` and `hasattr(mcp, "add_route")`
- If `mcp._app` is None/not available, need alternative mounting strategy

**Files**:
- Source: `/root/arifOS/arifosmcp/runtime/rest_routes.py`
- Container: `/usr/src/app/arifosmcp/runtime/rest_routes.py`
- Live container FS is read-only тАФ requires image rebuild to fix

**Key findings from debugging**:
- `IS_FASTMCP_3 = True`, `IS_FASTMCP_2 = False`
- `hasattr(mcp, "_app") = False` in the container
- `hasattr(mcp, "add_route") = False`
- `mcp.custom_route` exists and is callable
- `rest_routes.register_rest_routes` is a function (not FastMCP) that adds routes via `mcp.custom_route()` тАФ but it's a plain function returning None, not a method
- The function does `mcp.custom_route(f"/{section}")` for multiple sections тАФ these DO get registered on the FastMCP instance
- The actual registration is via `mcp.custom_route(f"/{section}", methods=["GET"])` patterns inside `register_rest_routes(mcp, tool_registry)`
- But `register_rest_routes` is called with positional args inside `server.py` тАФ could be arg mismatch

**Traefik now proxies to nginx, which proxies to uvicorn тАФ routes work via nginx pass-through for existing routes (MCP, WebMCP)**. New routes need either:
1. Nginx pass-through (already added in nginx.conf) тАФ or
2. Direct registration on FastMCP via `custom_route`
